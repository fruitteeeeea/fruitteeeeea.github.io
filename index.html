重点项目介绍

下面的内容适合作为面试官重点浏览区域：每个项目都拆成玩法设计、程序设计与视觉设计三个角度。

ProjectRogueBall
建议替换为战斗截图 / 道具选择截图
2D Pixel Roguelike

ProjectRogueBall

弹珠肉鸽游戏。玩家击碎不断下落的方块敌人，在局内升级弹珠技能、获取装备、种植与收割农作物，最终挑战关底 Boss。

玩法设计将弹珠打砖块与肉鸽道具选择结合，缩短单局节奏，让对局更适合快速体验。
程序设计使用 ImGui 调试面板实时观察伤害、成长曲线与压力比例，辅助数值调整。
道具系统通过触发系统与钩子节点实现不同阶段的道具加成，方便扩展更多效果。
ProjectTheBazzel
建议替换为背包 / 地图 / 战斗截图
Backpack + Pachinko + Turn-based

ProjectTheBazzel

一个结合《杀戮尖塔》路线探索、《大巴扎》和《背包英雄》式背包管理的弹珠游戏。玩家通过路线选择、道具摆放与回合动作触发不同玩法流派。

路线探索提供分支路线，让玩家根据风险和奖励选择下一步目标。
背包管理设计 10 种道具与多种组合方式，让物品位置与构筑策略产生关联。
流程控制使用状态机管理回合制游戏流程，并用 ImGui 实时调试道具数值。
ProjectDogFight
建议替换为 HUD / 战斗截图
2D Touch Arcade

ProjectDogFight

触屏街机式飞机狗斗游戏。核心目标是在限定时间内存活，并通过高速机动、敌机追逐与战斗反馈营造挑战和爽感。

角色状态机用状态机管理关闭引擎、开启引擎、加速等操作，让移动逻辑更清晰。
敌人行为树敌人会根据当前局势做出追击、横移、攻击等不同应对。
视觉表现使用矢量图形、扁平化 HUD、粒子特效与故障艺术塑造复古未来感。
ProjectDrill
建议替换为钻地 / 关卡截图
2D Platformer

ProjectDrill

一个以“钻入特定土壤”为差异化机制的平台跳跃游戏。玩家在地面、泥土和冲刺等状态之间切换，探索关卡并处理敌人。

核心机制玩家使用钻头进入特殊地形，改变常规平台跳跃的移动路径。
状态机管理地面、泥土、冲刺等状态，减少复杂移动逻辑之间的耦合。
美术反馈通过粒子特效区分钻入、攻击敌人和拾取金币等关键行为。

游戏原型

这些原型用于验证不同玩法方向、关卡结构与系统设计。

FirstPersonDungeon

第一人称地牢原型。参考《塞尔达传说：时之笛》和《黑暗之魂》的环状迷宫结构，使用白盒关卡搭建并填充美术素材。

第一人称迷宫设计白盒关卡

ProjectDrift

3D 街机漂移游戏原型。玩家通过漂移积累更高分数，重点验证车辆手感与得分反馈。

3D漂移街机

ProjectFactor

等距视角运输线游戏原型，探索资源运输、线路布局和空间规划的玩法可能。

等距视角运输线规划

ProjectJumper

竖屏平台跳跃原型，模仿 Downwell 式的纵向推进体验，适合继续扩展为移动端轻量游戏。

竖屏平台跳跃移动端

Game Jam 作品

Game Jam 作品均在限定时间内围绕主题独立完成，覆盖玩法、美术与程序。

GMTK 2024 · Scale

Scale the Nuts

玩家需要躲避从天而降的坚果，并在限定时间内到达屏幕顶端。为了契合主题，玩家拥有一把可以放大或缩小坚果的枪，用尺寸变化创造生存空间。

Loop Working
